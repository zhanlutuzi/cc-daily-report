现在是早上的回顾环节。请帮我回顾昨天到今天做过的事情，然后一起讨论下一步。

## 步骤

### 1. 加载昨日摘要

用 Bash 读 `~/.claude/summaries/` 下最近 24-36 小时的所有摘要卡片（文件名按时间倒序）：

```bash
find ~/.claude/summaries -name "*.md" -mtime -1.5 -type f | sort -r
```

然后逐个 Read 这些文件。每个文件 frontmatter 里有 `source`（compact/manual/cron）、`project`、`timestamp`、`cwd`，正文是当时的上下文摘要。

### 2. 生成「昨日回顾」

基于读到的摘要卡片，用中文生成结构化回顾：

**🌅 昨日回顾**（按时间或项目组织）

- **做过的事**：按项目分组，每个项目列出关键进展（要点列表，引用具体文件路径）
- **关键决策**：昨天做出了哪些选择，为什么
- **未完成/卡住的**：哪些事没做完、哪些卡住了
- **遗留风险**：坑点、需要注意的边界

### 3. 引导讨论下一步

在回顾末尾，结合我下面的备注（如果有），主动提问 2-3 个关于下一步的问题，比如：
- 「昨天 X 没做完，今天打算继续还是先放一放？」
- 「Y 这个决策今天要落地吗？」
- 「有没有新的事情要插进来？」

问题要具体、可决策，不要泛泛而谈。

### 4. 处理备注

如果 `$ARGUMENTS` 非空，把它当作我今天想关注的方向，在引导提问时优先围绕这个方向展开。

### 5. 如果没有摘要

如果 `~/.claude/summaries/` 下没有近 36h 的摘要卡片，直接告诉我「昨天没有摘要卡片记录」，然后用一句话问我昨天做了什么，准备从我的回答开始今天的讨论。

## 注意

- 不要写客套话，直接进入回顾。
- 摘要卡片是已经提炼过的，不要照抄，要二次提炼。
- 时间范围用 36h 而不是 24h，避免错过昨晚晚些时候的摘要。
